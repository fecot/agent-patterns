# 記事テンプレート

記事化するときの骨子。devlog の内容をこの順に流し込む。

## タイトル案
（例:「業務エージェント開発を学ぶ — Phase X: ○○ をどう作ったか」）

## 1. 導入（なぜこの Phase が必要か）
- この回で何ができるようになるか
- 業務エージェント全体の中でこの技術がどこに位置するか

## 2. 背景・設計判断
- 採用した構成と、その理由
- あえて採用しなかった選択肢とトレードオフ

## 3. 実装
- コードの要点（長すぎる貼り付けは避け、勘所だけ）
- ファイル構成と責務

## 4. ハマったポイント
- つまずいた点と解決
- 学習者が同じ轍を踏まないための注意

## 5. 動かしてみる
- 試せるコマンド
- 期待出力 / スクリーンショットの説明

## 6. まとめと次回
- この Phase で身についたこと
- 次の Phase への布石
